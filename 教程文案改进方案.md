# DeepScientist 教程文案改进方案（精简版）

## 一、核心问题

### 当前文案的问题
- **术语过于专业**："工作区"、"启动流程"、"运维控制中心"对新手不友好
- **缺少引导感**：像说明书而非教程，缺少"为什么"和"怎么做"
- **认知负担重**：一次性抛出太多概念
- **中英文风格不统一**：英文更简洁，中文有时过于正式

### 改进目标
- 让高中生也能轻松理解
- 像游戏教程一样循序渐进
- 保持专业但不失亲切
- 中英文各有特色但传达相同信息

---

## 二、改写原则

### 语言风格
1. **用类比降低门槛**：用熟悉的概念解释陌生的功能
2. **渐进式揭示**：先说"是什么"，再说"为什么"，最后说"怎么用"
3. **适度鼓励**：在关键步骤给予信心
4. **emoji极简使用**：仅在标题或关键提示处使用（每段最多1个）

### 中英文差异
- **中文**：可以更口语化，多用比喻，篇幅稍长
- **英文**：保持简洁直接，少用修饰，重点突出

### 3.5 研究目标（dialog-goal）

**原文（中文）：**
```
这里仍然是整个表单里**最重要的一格**。
尽量朴素地写清楚三件事：
1. 你到底想研究什么
2. 怎样才算成功
3. 在宣称进展前必须检查什么
```

**原文（英文）：**
```
This is still the **most important box** in the whole form.
Write three things as plainly as you can:
1. what you want to investigate
2. what would count as success
3. what must be checked before the system claims progress
```

**改写建议（中文）：**
```
最重要的一步：说清楚你的研究目标

用简单的话说清楚三件事：

1. 我想研究什么问题？
2. 什么结果算成功？
3. 需要验证哪些条件？

好的例子：
"复现 ResNet 在 CIFAR-10 上的结果，与论文的 baseline 对比，确认在单张 24GB GPU 上能训练完成。"

不好的例子：
"研究深度学习"（太模糊了）

提示：写得越具体，AI 就越能帮到你。
```

**改写建议（英文）：**
```
Most important step: Define your research goal clearly

State three things in plain language:

1. What problem do I want to investigate?
2. What result counts as success?
3. What conditions must be verified?

Good example:
"Reproduce ResNet on CIFAR-10, compare against the paper's baseline, and confirm it trains on a single 24GB GPU."

Bad example:
"Study deep learning" (too vague)

Tip: The more specific you are, the better the AI can help.
```

**原文（中文）：**
```

在这里，项目不再像一段聊天记录，而开始像一个真正有结构的科研系统。
```

**原文（英文）：**
```
Canvas makes the research map visible.

```

**改写建议（中文）：**
```
Canvas：你的研究地图

还记得游戏里的科技树吗？这就是你的"研究科技树"。

你会看到：
- 起点：最初的 baseline（基准实验）

- 失败路径：试过但没成功的方向（也很重要）
- 主干：最终成功的路线
- 成果：论文、分析、图表

为什么要看地图？
因为它告诉你这个项目是"怎么长出来的"，而不只是"最后停在哪里"。

提示：每个节点都是一次探索，失败的路径也有价值。
```

**改写建议（英文）：**
```
Canvas: Your research map

Think of this as your research tech tree.

You'll see:
- Starting point: The initial baseline
- Branches: Different ideas you tried
- Failed paths: Attempts that didn't work (still valuable)
- Main trunk: The successful route
- Outcomes: Papers, analysis, figures

Why look at the map?
It shows you how the project grew, not just where it ended.

Tip: Every node is an exploration—even failed paths teach you something.
```

### 3.7 Memory 记忆系统（workspace-memory）

**原文（中文）：**
```
Memory 是让项目持续生长的核心。
如果没有这样的界面，每一轮都可能像一次性消耗品
```

**原文（英文）：**
```
Memory is what lets the project keep growing.
Without a surface like this, each round risks becoming disposable.
```

**改写建议（中文）：**
```
Memory：你的研究笔记本

想象一下，如果每次玩游戏都没有存档，是不是很崩溃？Memory 就是你的"研究存档"。

它会记住什么？
- 成功的经验："这个方法有效，可以继续用"
- 失败的教训："这条路走不通，别再试了"
- 重要的发现："这组参数效果最好"

为什么重要？
在长期项目中，它决定了你是在持续进步，还是在原地打转。

提示：定期查看 Memory，避免重复踩坑。
```

**改写建议（英文）：**
```
Memory: Your research notebook

Imagine playing a game with no save file—frustrating, right? Memory is your "research save file."

What does it remember?
- Successful approaches: "This method worked, keep using it"
- Failed attempts: "This path didn't work, don't retry"
- Key findings: "These parameters gave the best results"

Why does it matter?
In long projects, it's the difference between steady progress and going in circles.

Tip: Check Memory regularly to avoid repeating mistakes.
```

## 四、实施计划

### 改写优先级
1. **核心流程**（高优先级）：首页、创建对话框、Canvas、Memory、Copilot
2. **辅助功能**（中优先级）：BenchStore、导航栏、文件浏览
3. **高级功能**（低优先级）：设置页面、管理界面

### 改写要点总结
- **减少专业术语**：用"研究地图"代替"项目结构"，用"任务商店"代替"BenchStore"
- **增加类比**：游戏科技树、存档系统、NPC助手等
- **渐进式引导**：先说是什么，再说为什么，最后说怎么用
- **适度鼓励**：在关键步骤给予信心，降低焦虑感
- **emoji极简**：仅在标题使用，正文不用

### 中英文差异处理
- **中文**：可以更口语化，多用"就像..."的类比，篇幅稍长
- **英文**：保持简洁，用"Think of it as..."，重点突出
